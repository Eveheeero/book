9.70 KScratchn Registers (CP0 Register 31, Selects 2 to 7) MIPS® Architecture For Programmers Vol. III: MIPS32® / microMIPS32™ Privileged Resource Architecture, Rev. 6.02 311 9.70 KScratchn Registers (CP0 Register 31, Selects 2 to 7) Compliance Level: Pre-Release 6 - Optional, KScratch1 and KScratch2 at selects 2, 3 are recommended. Release 6 - Required. The KScratchn registers are read/write registers available for scratch pad storage by kernel mode software. These reg- isters are 32bits in width for 32-bit processors and 64bits for 64-bit processors. The existence of these registers is indicated by the KScrExist field within the Config4 register. The KScrExist field specifies which of the selects are populated with a kernel scratch register. Debug Mode software should not use these registers, instead debug software should use the DESAVE register. If EJTAG is implemented, select 0 should not be used for a KScratch register. Select 1 is being reserved for future debug use and should not be used for a KScratch register.
