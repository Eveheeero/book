9.39 Exception Program Counter (CP0 Register 14, Select 0) MIPS® Architecture For Programmers Volume III: MIPS64® / microMIPS64™ Privileged Resource Architecture, Rev. 6.03 243 9.39 Exception Program Counter (CP0 Register 14, Select 0) Compliance Level: Required. The Exception Program Counter (EPC) is a read/write register that contains the address at which processing resumes after an exception has been serviced. All bits of the EPC register are significant and must be writable. Unless the EXL bit in the Status register is already a 1, the processor writes the EPC register when an exception occurs. • For synchronous (precise) exceptions, EPC contains either: • the virtual address of the instruction that was the direct cause of the exception, or • the virtual address of the immediately preceding branch or jump instruction, when the exception causing instruction is in a branch delay slot, and the Branch Delay bit in the Cause register is set. • For asynchronous (imprecise) exceptions, EPC contains the address of the instruction at which to resume execu- tion. The processor reads the EPC register as the result of execution of the ERET instruction. Software may write the EPC register to change the processor resume address and read the EPC register to determine at what address the processor will resume.
