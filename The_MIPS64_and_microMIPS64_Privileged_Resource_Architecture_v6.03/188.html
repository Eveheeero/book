With this additional level, the length of the page table walk increases to 5 levels from 4. The PWCtl PWDirExt field is used by Software to determine the presence of this feature. PTEW 5..0 Specifies the left shift applied to the Page Table index, in addition to the shift required to account for the native data size of the machine. The set of available shifts is implementation-dependent. Software can discover the available values by writing this field. If the requested shift value is not available, PTEW will be written as zero. A shift of one must be imple- mented. In Release 6, if PWSize PS = 1, then it is implementation- dependent as to whether PTEW is writeable, or read-only 0. PTE sizes of 128-bits are not recommended.
