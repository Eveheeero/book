In Release 5 with EVA, and Release 6 onwards, access to 64-bit Supervisor Segment is prevented by explicitly sign-extending the address from bit 31 when KX=0 or SX=0, providing the inputs that generate the address are also sign-extended from bit 31.See “32-bit Compatibility Addressing for Release 5 EVA, and Release 6” on page 39 and Table 9.50 for further detail. In Release 5 with EVA, and Release 6 onwards, if KX is written as 0, then SX is also written as 0. Supervisor can always execute 64-bit instructions, regard- less of the state of SX. Prior to Release 6, if Supervisor Mode is not implemented, it is implementation-dependent whether access to what would normally be 64-bit supervisor address space is enabled with the SX or KX bit. In Release 6, if Supervisor Mode is not implemented, access to what would normally be 64-bit supervisor address space is enabled with the SX bit. If 64-bit addressing is not implemented, this bit must be ignored on write and read as zero.
