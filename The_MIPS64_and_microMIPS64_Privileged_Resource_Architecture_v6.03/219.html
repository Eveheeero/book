In implementations of Release 2 of the Architecture in which EIC interrupt mode is enabled (Config3 VEIC = 1), these bits are writable, but have no effect on the interrupt system. R/W Undefined Required KX 7 Enables the following behavior: • Access to 64-bit Kernel Segments. • Use of the XTLB Refill Vector for references to Kernel Segments. Prior to Release 5, or in Release 5 without Enhanced Vir- tual Addressing (EVA), access to 64-bit Kernel Segments is disabled, when KX=0, by causing an Address Error. In Release 5 with EVA, and Release 6 onwards, access to 64- bit Kernel Segments is prevented by explicitly sign- extending the address from bit 31 when KX=0, providing the inputs that generate the address are also sign-extended from bit 31. See “32-bit Compatibility Addressing for Release 5 EVA, and Release 6” on page 39 and Table 9.50 for further detail. Kernel mode can always execute 64-bit instructions, regardless of the state of KX. If 64-bit addressing is not implemented, this bit must be ignored on write and read as zero.
