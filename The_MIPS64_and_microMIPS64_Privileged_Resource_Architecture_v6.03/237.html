This bit both indicates that the watch exception was deferred, and causes the exception to be initiated once Status EXL and Status ERL are both zero. As such, software must clear this bit as part of the watch exception handler to prevent a watch exception loop. For Pre-Release 6, software should not write a 1 to this bit when its value is a 0, thereby causing a 0-to-1 transition. If such a transition is caused by software, it is UNPRE- DICTABLE whether hardware ignores the write, accepts the write with no side effects, or accepts the write and ini- tiates a watch exception once Status EXL and Status ERL are both zero. For Release 6, hardware ignores a write of 1. If watch registers are not implemented, this bit must be ignored on write and read as zero.
