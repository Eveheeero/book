300 MIPS® Architecture For Programmers Volume III: MIPS64® / microMIPS64™ Privileged Resource Architecture, Revision 6.03 Software must follow the described method for reprogramming the attribute state of a MAAR pair. • Disable the MAAR pair by clearing MAAR VL . Accesses to the MAAR region become non-speculative. • Program PageGrain ELPA as needed. • Set MAAR VL along with other fields in MAAR. 32-bit software uses MTC0, which initializes the lower half of MAAR. 64-bit software uses DMTC0 to initialize the entire MAAR. • Initialize MAAR[63:32], which is present if PageGrain ELPA =1. This includes MAAR VH , if Config5 MVH =1. Figure 9.58 shows the format of MAAR when PageGrain ELPA =0 i.e., MAAR is limited to a 32-bit register. Figure 9-59 shows the format of MAAR when PageGrain ELPA =1 and Config5 MVH =0 i.e., MAAR is a 64-bit register accessible only by DMTC0 and DMFC0, and thus the upper valid bit is not present. Figure 9-60 shows the format of MAAR when PageGrain ELPA =1 and Config5 MVH =1 i.e., MAAR is a 64-bit register accessible not only by DMTC0 and DMFC0, but MTHC0 and MFHC0, for 32-bit backward compatibility, and thus the upper valid bit is present.
