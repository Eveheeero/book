This field contains least-significant bits of the physical page number corresponding to the virtual page. If the processor is enabled to support large physical addresses, the PFNX field, described above is concatenated with the PFN field to form the full page frame number. If the processor is not enabled to support large physical addresses, the entire page frame number is represented by this field. See the description of the PFNX field above for more information. If the processor is enabled to support 1 kB pages (Config3 SP = 1 and PageGrain ESP = 1), the PFN field corresponds to bits 33..10 of the physical address (the field is shifted left by 2 bits relative to the Release 1 def- inition to make room for PA 11 10 ). If the processor is not enabled to support 1 kB pages (Config3 SP = 0 or PageGrain ESP = 0), the PFN field corresponds to bits 35..12 of the physical address.
