Virtual Memory 40 MIPS® Architecture For Programmers Volume III: MIPS64® / microMIPS64™ Privileged Resource Architecture, Rev. 6.03 Instruction fetch and Hardware Page Table Walker address generation includes speculative non-instruction based addressing to which the 32-bit compatibility rules also apply. For speculative instruction fetch the rules for addressing due to sequential fetch and control-flow changes are identical to the rules for control flow instruction execution : the fetch PC must be checked for Address Error, and in its absence, the PC is sign-extended. Branch instructions and PC-relative Release 6 ADDIUPC, ALUIPC, AUIPC, LWPC, LWPUPC, LDPC do not require canonical checks of the inputs to address generation, as the PC itself is typically always canonically compli- ant. However, it is required that the function effective_address() applies to such instructions. PC-Region Jumps also fall into the same category of PC-relative instructions. Software may errantly cause the PC to be canonically non-compliant by writing an inappropriate value to CP0 EPC, or writing a value to Status KX/SX/UX that is incompatible with the current state of PC. For the latter case, software must execute an instruction hazard barrier (e.g., JALR.HB) after the MTC0 that changes Status KX/SX/UX so the change takes effect no later than the target of the hazard barrier. Application of effective_address() will take care of the former case. The pseudo-code for effective_address describes the new scheme for effective address generation for 32-bit compati- ble addressing. The pseudo-code is applied specifically to instruction and data addressing instructions.
