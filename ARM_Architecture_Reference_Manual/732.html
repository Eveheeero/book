ARM DDI 0100I Second-level page table descriptor fields The fields in a second-level page table have the following meanings: Bits[1:0] Identify the type of descriptor (and include XN bit in revised VMSAv6 format). Bits[3:2] Are the cacheable and bufferable bits. Bits[5:4] Are the access permission bits, full page or AP0 subpage. The following bits are used for the corresponding physical address bits, the field size depending on the page size: Bits[31:16] large (64KB) pages Bits[31:12] small (4KB) pages The following bits are used for additional access control functions: Bits[15:6] depending on the format: • large page control • subpage access permissions • TEX • APX • S • nG • XN Bits[11:6] depending on the format: • small page control • subpage access permissions • TEX • APX • S • nG Bits[9:6] tiny page control, SBZ. For details of these fields see the following sections: AP and APX see Access permissions on page B4-8. C, B and TEX see C, B, and TEX Encodings on page B4-11 XN, nG and S see Page table translation in VMSAv6 on page B4-25. Where subpages are supported, the page is divided into four blocks, each of the same size. AP0 refers to the block with the lowest block base address, with AP1, AP2 and AP3 applying to blocks with incrementing block base addresses.
