ARM DDI 0100I B5.1 About the PMSA The MPU based Protected Memory System Architecture (PMSA) provides a considerably simpler memory protection scheme than the MMU based model described in Chapter B4 Virtual Memory System Architecture. The simplification applies to both the hardware and the software. The main simplification is that the MPU does not use translation tables. Instead, System Control Processor (CP15) registers are used to fully define protection regions, eliminating the need for hardware to do translation table walks, and for software to set up and maintain the translation tables. This has the benefit of making the memory checking fully deterministic. However, the level of control is now region based rather than page based, that is, the control is considerably less fine-grained. A second simplification is that virtual-to-physical address mapping is not supported. The physical memory address is always the same as the virtual address generated by the ARM ® processor. The following features are common to all PMSA designs: • The memory is divided into regions. System Coprocessor registers are used to define the region size, base address, and memory attributes, for example, cacheability, bufferability and access permissions of a region. • Memory region control (read and write access) is permitted only from privileged modes. • If an address is defined in multiple regions, a fixed priority scheme (highest region number) is used to define the properties of the address being accessed. • An access to an address that is not defined in any region causes a memory abort. • All addresses are physical addresses, address translation is not supported. • Support for unified (von Neumann) and separate (Harvard) instruction and data address spaces.
