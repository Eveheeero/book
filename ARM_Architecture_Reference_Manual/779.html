B5-27 B5.7.9 Fault address (PMSAv6) When the CRm field is c0, register 6 enables the data and instruction Fault Address registers to be accessed, depending on the value of the Opcode_2 field. Note The WFAR feature is migrating from CP15 to the debug architecture in CP14 and as such decoding the WFAR through CP15 is deprecated in ARMv6. See Coprocessor 14 debug registers on page D3-2 for its revised location. CRm values of c1 and c2 are used to configure the memory region attributes as defined in Register 6: Memory region programming (PMSAv6) on page B5-28. The FAR and IFAR are updated on an abort in accordance with Table B5-16 on page B5-28. Writing CP15 register 6 enables the values of the FAR and IFAR to be written. This is useful for a debugger to restore their values. The WFAR is updated on a debug event in monitor mode. If the watchpoint was taken whilst in ARM state, the WFAR will contain the address of the instruction when the event happened + 0x8.
