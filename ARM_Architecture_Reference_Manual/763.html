B5-11 For an explanation of Normal, Strongly ordered and Device memory types along with the shareable attribute, see ARMv6 memory attributes - introduction on page B2-8. where s is the value of the S bit in the memory attribute register The terms Inner and Outer refer to levels of caches that might be built in a system. Inner refers to the innermost caches, including Level 1. Outer refers to the outermost caches. The boundary between inner and outer caches is defined in the implementation of a cached system. Inner always includes L1. For example, in a system with three levels of cache, the Inner attributes might apply to L1 and L2, and the Outer attributes to L3. In a system with two levels, it is envisaged that Inner attributes would be applicable to L1 and Outer attributes to L2.
