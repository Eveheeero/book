x87 Floating-Point FWAIT (WAIT) 321 Instruction Reference 26569—Rev. 3.16—November 2021 AMD64 Technology Forces the processor to test for pending unmasked floating-point exceptions before proceeding. If there is a pending floating-point exception and CR0.NE = 1, a numeric exception (#MF) is generated. If there is a pending floating-point exception and CR0.NE = 0, FWAIT asserts the FERR output signal, then waits for an external interrupt. This instruction is useful for insuring that unmasked floating-point exceptions are handled before altering the results of a floating point instruction. FWAIT and WAIT are synonyms for the same opcode.
