252 General-Purpose Instruction Reference AMD64 Technology 24594—Rev. 3.35—June 2023 Copies the 32-bit value in a register or memory location (second operand) into a 64-bit register (first operand), extending the most significant bit of the 32-bit value into all higher bits of the 64-bit register. This instruction requires the REX prefix 64-bit operand size bit (REX.W) to be set to 1 to sign-extend a 32-bit source operand to a 64-bit result. Without the REX operand-size prefix, the operand size will be 32 bits, the default for 64-bit mode, and the source is zero-extended into a 64-bit register. With a 16- bit operand size, only 16 bits are copied, without modifying the upper 48 bits in the destination. This instruction is available only in 64-bit mode. In legacy or compatibility mode this opcode is interpreted as ARPL.
