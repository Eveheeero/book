Refer to “Vectors” on page 244 for a complete description of each interrupt, and a description of the interrupt mechanism. 1.7 Additional System-Programming Facilities 1.7.1 Hardware Multitasking A task is any program that the processor can execute, suspend, and later resume executing at the point of suspension. During the time a task is suspended, other tasks are allowed to execute. Each task has its own execution space, consisting of a code segment, data segments, and a stack segment for each privilege level. Tasks can also have their own virtual-memory environment managed by the page- translation mechanism. The state information defining this execution space is stored in the task-state segment (TSS) maintained for each task. Table 1-2.
