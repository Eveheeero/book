Each type of x87 floating-point exception can be masked by setting the appropriate bits in the x87 control-word register. See “x87 Floating-Point Programming” in Volume 1 for more information on using this register. #MF can also be disabled by clearing the CR0.NE bit to 0. See Section “Numeric Error (NE) Bit” on page 43 for more information on using this bit. 8.2.17 #AC—Alignment-Check Exception (Vector 17) An #AC exception occurs when an unaligned-memory data reference is performed while alignment checking is enabled. After a processor reset, #AC exceptions are disabled. Software enables the #AC exception by setting the following register bits: • CR0.AM=1. • RFLAGS.AC=1. When the above register bits are set, an #AC can occur only when CPL=3. #AC never occurs when CPL 3.
