The instruction following the MOV CR0 that enables paging must be a branch, and both the MOV CR0 and the following branch instruction must be located in an identity-mapped page. 14.6.2 Consistency Checks The processor performs long-mode consistency checks whenever software attempts to modify any of the control bits directly involved in activating long mode (EFER.LME, CR0.PG, and CR4.PAE). A general-protection exception (#GP) occurs when a consistency check fails. Long-mode consistency checks ensure that the processor does not enter an undefined mode or state that results in unpredictable behavior. Long-mode consistency checks cause a general-protection exception (#GP) to occur if: • An attempt is made to enable or disable long mode while paging is enabled. • Long mode is enabled, and an attempt is made to enable paging before enabling physical-address extensions (PAE). • Long mode is enabled, and an attempt is made to enable paging while CS.L=1. • Long mode is active and an attempt is made to disable physical-address extensions (PAE).
