This is illustrated in the figure below. Figure 15-15. vAPIC Backing Page Access To correctly redirect guest accesses of the guest vAPIC registers to the vAPIC backing page, the hard- ware needs two addresses. These are: • vAPIC backing page address in the SPA space • Guest vAPIC base address (APIC BAR) in the GPA space System software is responsible for setting up a translation in the nested page table granting guest read and write permissions for accesses to the vAPIC Backing Page in SPA space. AVIC hardware walks the nested page table to check permissions, but does not use the SPA address specified in the leaf page table entry.
