However, these would be the only instructions with partial register writes and would add com- plexity in implementations with recoded ﬂoating-point or register renaming, requiring a pipeline read-modify-write sequence. Scaling up to handling quad-precision for RV32 and RV64 would also require additional instructions if they were to follow this pattern. The ISA was deﬁned to reduce the number of explicit int-ﬂoat register moves, by having conversions and comparisons write results to the appropriate register ﬁle, so we expect the beneﬁt of these instructions to be lower than for other ISAs.
